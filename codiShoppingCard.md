# Què és una Pull Request?

Una **Pull Request (PR)** és una sol·licitud formal perquè els canvis realitzats en una branca d’un repositori siguin revisats i fusionats amb una altra branca, habitualment la branca principal (*main* o *master*). És una funcionalitat present a plataformes com **GitHub**, **GitLab** o **Bitbucket**.

## Per a què serveix?

- Permet notificar als col·laboradors que s’han realitzat canvis.
- Facilita la revisió de codi (*code review*).
- Ajuda a garantir que les noves funcionalitats o correccions no trenquin el projecte.
- Centralitza la discussió i el seguiment dels canvis abans de fusionar-los.

## Avantatges d’una Pull Request

- **Revisió col·laborativa:** Els membres de l’equip poden comentar, proposar millores i detectar errors.
- **Històric clar:** Queda un registre detallat de què s’ha canviat i per què.
- **Control de qualitat:** Permet integrar tests automàtics, integració contínua (CI) i altres verificacions abans de fusionar.
- **Seguretat:** Evita que canvis no revisats o potencialment problemàtics arribin a la branca principal.

## Desavantatges d’una Pull Request

- **Pot afegir lentitud:** Si la revisió triga massa, pot frenar el desenvolupament.
- **Excés de burocràcia:** En projectes petits pot resultar una càrrega innecessària.
- **Conflictes freqüents:** Si molts desenvolupadors treballen en paral·lel, poden aparèixer conflictes de fusió que requereixin resolució manual.
- **Debats llargs:** A vegades la discussió al voltant del codi pot allargar-se més del necessari.
